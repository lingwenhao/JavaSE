命令模式(Commande):
    描述:
        -将命令的请求者和执行者解耦，通过命令关联起来
        -在命令者模式中,会为一个请求封装一个对象,以便不同的参数来表示不同的请求(即命令)
        -命令模式必须支持撤回操作
    角色分析:
        -调用者(Invoker):
        -命令者(Command):,需要执行的所有接口都在这里,可以是接口或抽象类
        -接收者(Receiver):,知道如何实施和执行一个请求相关的操作
        -具体的命令执行者(ConcreteCommad):将一个接收者对象与一个动作,调用接收者的相关操作,实现execute
    好处:
        -让对象之间的调用关系更加灵活，实现解耦
